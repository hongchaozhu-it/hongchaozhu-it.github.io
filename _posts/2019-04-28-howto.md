这个主题的上手方式并不复杂。你只需要把文章放进 `_posts`，把独立页面放进 `_pages`，再在 `_config.yml` 里补齐站点信息，就能得到一个结构完整的中文站点。

一个常见的开始流程如下：

1. 修改站点标题、描述、作者和部署地址。
2. 删除默认示例文章，换成自己的中文内容。
3. 根据需要保留 About、Contact 这类独立页面。
4. 本地预览确认排版正常后，再部署到 GitHub Pages。

最基础的配置通常长这样：

```yaml
title:            山海手记
longtitle:        山海手记 · 中文知识手册
author:           你的名字
description: >
  这里写站点简介。
```

如果你准备把这里当成一个持续更新的内容站，建议尽早约定文章命名方式、分类习惯和封面图规范，这样后面越写越顺手。

> ##### 提示
>
> 与传统静态生成流程相比，这种方式最大的好处是：你平时只需要维护 Markdown 源文件，不需要把生成后的 HTML 一起提交到仓库。
